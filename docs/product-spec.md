# RecallPlate — Product Specification

## Overview

RecallPlate is a consumer-facing web application that combines FDA and USDA food recall data into a single, clean, searchable dashboard. It is the "weather app for food safety" — a public utility that helps parents, families, and consumers quickly answer: **"Is my food safe?"**

No one has built a modern, well-designed product that merges both data sources (openFDA + USDA FSIS) into a unified experience. The existing landscape is government websites from 2004, a basic mobile app, and abandoned GitHub repos. RecallPlate fills that gap.

---

## Target Audience

**Primary:** Consumers and parents ("Is this product recalled? What's been recalled in my state?")

**Secondary (future phases):** Journalists/media, food industry professionals, personal injury attorneys, insurance underwriters

## Data Sources

### 1. openFDA — Food Enforcement Reports API
- **Endpoint:** `https://api.fda.gov/food/enforcement.json`
- **Coverage:** FDA-regulated food products (everything except meat, poultry, and egg products)
- **History:** 2004–present, updated weekly
- **Key fields:** `recall_number`, `classification` (Class I/II/III), `product_description`, `reason_for_recall`, `recalling_firm`, `distribution_pattern`, `state`, `city`, `report_date`, `recall_initiation_date`, `status`, `product_type`
- **Rate limits:** 240 requests/minute with API key (free), 40/minute without
- **Docs:** https://open.fda.gov/apis/food/enforcement/

### 2. USDA FSIS — Recall API
- **Endpoint:** `https://www.fsis.usda.gov/api/recall-api` (or scrape from `fsis.usda.gov/recalls`)
- **Coverage:** Meat, poultry, and egg products
- **Key fields:** Recall number, classification (Class I/II/III), product name, reason, company, distribution states, date, pounds recalled
- **Note:** The USDA API is less structured than openFDA. Distribution state data often requires parsing free-text fields. Plan for normalization logic.
- **Docs:** https://www.fsis.usda.gov/science-data/data-sets-visualizations

### 3. openFDA — Animal & Veterinary Adverse Events (Pet Food)
- **Endpoint:** `https://api.fda.gov/animalandveterinary/enforcement.json`
- **Coverage:** Pet food and animal feed recalls
- **History:** Same as openFDA food enforcement (2004–present)
- **Key fields:** Same structure as food enforcement reports
- **Note:** Normalize into the same unified schema with `product_category: "Pet Food"`

### Data Normalization

Both sources must be normalized into a unified schema:

```
RecallEvent {
  id: string                    // Internal unique ID
  source: "FDA" | "USDA"       // Origin agency
  recall_number: string         // Original recall number
  classification: "I" | "II" | "III"  // Severity class
  status: string                // Ongoing, Completed, Terminated
  product_description: string   // What was recalled
  product_category: string      // Normalized category (see below)
  reason: string                // Why it was recalled
  reason_category: string       // Normalized (contamination, allergen, foreign material, etc.)
  recalling_firm: string        // Company name
  distribution_states: string[] // Array of state abbreviations
  nationwide: boolean           // If distributed everywhere
  report_date: date             // When reported
  recall_initiation_date: date  // When recall started
  city: string                 // Recalling firm city
  state: string                // Recalling firm state
  quantity: string             // Amount recalled (pounds for USDA, units for FDA)
  url: string                  // Link to original source
  ai_summary: string           // Plain-English summary generated by Claude at ETL time
}
```

## Architecture

### Data Pipeline

```
[openFDA API] ──→ ETL/Normalize ──→ [Database] ──→ [Next.js API Routes] ──→ [Frontend]
[USDA FSIS API] ─→ ETL/Normalize ──↗
```

### Strategy: Cached Static Data with Scheduled Refresh

- **Database:** PostgreSQL (or SQLite for v1 prototype)
- **Sync schedule:** Cron job runs daily (or every 6 hours for v1.1)
- **ETL job:** Node.js script or serverless function that:
  1. Pulls latest data from both APIs (food + animal/veterinary)
  2. Normalizes into unified schema
  3. Categorizes products and reasons using keyword matching / simple rules
  4. Generates plain-English AI summary for each new recall via Claude Opus 4.6 (`claude-opus-4-6`) through the Anthropic API (batch at ETL time, not on-the-fly)
  5. Upserts into database
  6. Logs sync status and any parsing failures
- **Why not real-time:** Both APIs are rate-limited and slow. Pre-caching gives instant load times and lets us enrich/normalize the data.

### Tech Stack

- **Framework:** Next.js (App Router)
- **Language:** TypeScript
- **Database:** PostgreSQL (Supabase, Neon, or Railway)
- **ORM:** Prisma or Drizzle
- **Hosting:** Vercel
- **Styling:** Tailwind CSS
- **Charts:** Recharts or D3
- **Map:** Leaflet or react-simple-maps (US choropleth)
- **Search:** Full-text search via PostgreSQL or Algolia (later)

## V1 Feature Set

### 1. Dashboard (Home Page)

The default landing experience. Shows a high-level view of food recall activity.

**Components:**

#### a. Hero / Summary Stats
- Total active recalls (last 30 days)
- Breakdown by source (FDA vs USDA)
- Most common reason category this month
- "Last updated: [date/time]"

#### b. US Map Choropleth
- Color-coded by number of recalls affecting each state
- Click a state to filter all dashboard data to that state
- Toggle: last 30 days / 90 days / 12 months / all time

#### c. Timeline Chart
- Line or area chart showing recall volume over time
- Filterable by product category and severity
- Default view: last 12 months, monthly granularity

#### d. Product Category Breakdown
- Bar or horizontal bar chart
- Shows recall count by normalized product category
- Clickable to filter the recall list

#### e. Severity Distribution
- Visual breakdown of Class I / II / III recalls
- Simple donut or segmented bar
- With plain-English labels: "Serious health risk" / "Remote risk" / "Not likely harmful"

#### f. Recent Recalls Feed
- Scrollable list of most recent recalls
- Each card shows: product name, company, reason, severity badge, date, source badge (FDA/USDA)
- Click to expand full detail

### 2. Search

A dedicated search experience for answering "Is this product recalled?"

- **Text search:** Product name, brand name, company, keyword
- **Filters:**
  - State (dropdown or type-ahead)
  - Product category (multi-select)
  - Severity class (I, II, III)
  - Date range
  - Source (FDA, USDA, or both)
  - Reason category
- **Results:** Card list with recall details, sorted by date (newest first)
- **Empty state:** "No matching recalls found" with helpful guidance

### 3. Recall Detail Page

Each recall gets a permalink (`/recall/[id]`).

- Full product description
- **AI-generated plain-English summary** (2-3 sentences a parent can scan in 10 seconds: what product, what's wrong with it, what to do)
- Recalling firm and location
- Reason for recall (plain English + original text)
- Severity classification with explanation
- Distribution states (highlighted on a mini map)
- Date information (reported, initiated)
- Quantity recalled
- Link to original FDA/USDA source
- Related recalls (same company or same product category)

### 4. Alert System (Placeholder for v1)

v1 ships with a simple email capture:
- "Get notified about recalls in your state" — email + state dropdown
- Backend stores subscriber list
- Actual email delivery is v2

## Decisions Made

- **No Canadian recalls.** US-only (FDA + USDA).
- **Yes pet food in v1.** Include pet food recalls using openFDA's animal/veterinary enforcement endpoint. Add "Pet Food" as a product category in the normalized schema.
- **Yes AI summaries.** Each recall detail page should include a plain-English summary generated by Claude/LLM that translates the often-dense government recall language into something a parent can understand in 10 seconds. Generate at ETL time and store in the database (not on-the-fly).
